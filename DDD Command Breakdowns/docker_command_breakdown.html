Breaking Down the Command

docker run -d -p 8080:80 nginx

1. docker run

Starts a new container from a given image. In this case, we are running a container based on the nginx image.

2. -d (detached mode)

Runs the container in the background so your terminal is not blocked. Without -d, logs would stream directly into your terminal.

Run Modes

Foreground (default)
docker run nginx
Container runs in the foreground, logs stream directly to your terminal. You stop it with Ctrl+C.

Detached (-d)
docker run -d nginx
Container runs in the background, and you can continue using your terminal. Check logs with docker logs <container_id>.

3. -p 8080:80 (port binding)

Maps port 8080 on your host machine → to port 80 inside the container.

Think of it as a bridge: 
Browser → localhost:8080 → Docker Engine → Container:80 (Nginx)

4. nginx (image)

This is the Docker image to run. If not present locally, Docker will pull it from Docker Hub. By default, Nginx listens on port 80 inside the container.

✨ Summary

This command runs an Nginx web server container in the background, makes it accessible at http://localhost:8080, and forwards that traffic into the container’s port 80.
